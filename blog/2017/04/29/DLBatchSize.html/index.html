深度学习 之 batch_size 的设置

发表于 2017-04-29 | 分类于 Machine Learning ， Deep Learning | | 阅读次数

在深度神将网络训练过程中，针对数据量的大小，有两种训练方式，一种是全数据集（Full-batch Learning）载入训练，一种是小批量数据集（Mini-batch Learning）载入训练。

1、全数据集训练

在训练集数量比较小时，往往使用全数据集载入训练，这样有两个好处：

  • 全数据集确定的更新方向能偶更好地代表样本总体，从而更准确的朝向极值所在的方向；
  • 由于不同权重的梯度值差别较大，因此选取一个合适的全局学习率很困难，全数据集训练可以使用Rprop方法更新参数。

2、小批量数据集训练

在训练数据集数量十分庞大时，完成一次数据更新，需要很长的时间和空间（内存），上述两个优点又变成了缺点：

  • 在深度学习中，通常使用GPU进行模型训练，而GPU的内存一般都比较小，因此一次性载入所有训练数据，比较困难；
  • 以Rprop方式更新参数，会由于各个batch之间的采样差异性，各次梯度修正值相互抵消，无法修正，这才有了后来的RMSProp的妥协方案。

3、使用建议

在实际工程实践中，通常使用小批量数据梯度下降法（Miin-batch Gradient Descent），可以使用小批量数据替代总体数据的原因是，当训练集足够大时，训练集中的数据都是相关的，小批量数据的提督就是整体数据集梯度的一个近似值。

4、在合理范围内增大batch size的好处

  • 提高内存利用率
  • 跑完一次epoch(全数据集)所需的迭代次数减少，加快同一数据量的处理速度
  • 一般来说，batch size越大，其确定的参数更新方向越准，引起的训练震荡越小

5、盲目增大batch size的坏处

  • 容易导致内存溢出
  • 需要花费更多的时间达到相同的精度
