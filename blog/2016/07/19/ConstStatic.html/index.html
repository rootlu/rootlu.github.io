const与static

发表于 2016-07-19 | 分类于 C/C++ | 阅读次数

const和static是C++中的两种修饰符，const限定变量的值不可改变，即将变量定义为了一个常量，static修饰的变量则表示该变量是静态的，它被用来控制变量的存贮方式和可见性。

1. const

常量限定符，即const对象一旦创建其值就不能再改变，所以必须初始化
多个文件之间共享const对象，必须在变量定义之前添加extern关键字

1.2 修饰普通变量和指针

  • 关键字const出现在星号左边，表示被指物为常量；出现在右边，指针自身是常量；出现在两边，被指物和指针两者都是常量

1.3 修饰函数参数

  • 表示函数体内不能改变参数的值

1.4 修饰成员变量

  • 只能在初始化列表中赋值

1.5 修饰成员函数

  • 该成员函数可作用于const对象身上

1.6 尽可能的使用const，而不使用#define

  • 编译器处理方式不同，define宏在预处理阶段展开，const常量在编译运行阶段使用；
  • 类型和安全检查不同，define宏没有类型，不做任何类型检查，const常量有具体类型，在编译阶段进行类型检查
  • 存储方式不同，define宏不会分配内存，const常量会在内存中分配内存

2. static

C++ 修饰符 ，表示 “静态的”

2.1 作用：

  • 扩展生存周期
  • 限制作用域
  • 唯一性

2.2 用法

  • 类的静态成员

在cpp文件中必须对变量value初始化，int A::value=0，这里没有static修饰
类的静态成员是该类所有实例的共用成员，即该成员在类内是一个全局变量，只分配一次

  • 类的静态函数

函数实现不需要static修饰，类的静态函数在该类内是全局函数，不能访问类的私有成员，只能访问静态成员，不需要类的实例即可调用，增加了类的访问权限的全局函数
虚函数不能是静态的，静态成员函数可继承、覆盖

  • 静态全局变量
    该变量只在这一个cpp内有效，在其他cpp中不能访问
    声明所有cpp可共享的全局变量，在头文件中声明为extern的，在任意一个包含头文件的cpp中初始化一次就好
  • 静态全局函数
    在cpp内声明，函数实现不需要static修饰
