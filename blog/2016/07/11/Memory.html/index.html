C++内存管理

发表于 2016-07-11 | 分类于 C/C++ | | 阅读次数

内存泄漏几乎在每个C++程序中都会发生，如何管理好内存是一件令C++程序猿非常头疼的事情，稍有不慎就会收获意想不到的“惊(jing)喜(xia)”，因此要想驾驭好C++这匹野马，就必须充分理解运用好内存管理这一有力的缰绳。

1. 内存分配方式

举个栗子
int *p =new int [5];
new的是堆内存，指针分配的是栈内存
在栈内存放一个指向一块堆内存的指针p
记得释放内存 delete[]p

1.2 栈(stack) 堆栈简称栈

  • 函数内局部变量的存储单元
  • 函数执行结束自动释放
  • 效率高
  • 内存容量有限

1.3 堆(heap)

  • new/delete分配释放
  • 程序结束，操作系统自动回收

1.4 自由存储区

  • malloc/free分配释放

1.5全局/静态存储区

  • 全局变量和静态变量

1.6 常量存储区

  • 常量，不可更改

2. 堆和栈的区别

2.1 分配方式不同

  • 堆是动态分配（new/delete）
  • 栈有静态分配（编译器完成，局部变量）和动态分配（alloca函数，编译器释放）

2.2 分配效率不同

  • 堆由C/C++函数库提供，效率低
  • 栈由机器系统提供，效率高

2.3 管理方式不同

  • 堆由程序员控制 碎片
    • 栈由编译器自动管理

2.4 空间大小不同

  • 32位，堆内存可达4G
    • VC6 默认栈1M，可改

2.5 碎片问题

  • 堆有碎片
  • 栈无碎片，先进后出

2.6 生长方向不同

  • 堆 向上生长，向着内存地址增加方向
  • 栈 向下生长，向着内存地址减小方向
