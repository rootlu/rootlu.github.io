TCP/IP协议

发表于 2016-04-10   |   分类于 NetWork   |     |   阅读次数

TCP/IP协议族（TCP/IP Protocol Suite），简称TCP/IP，这个协议家族有两个核心协议，包括TCP（传输控制协议）和IP（网际协议）。TCP/IP提供点对点的链接机制，将数据应该如何封装、定址、传输、路由以及在目的地如何接收，都加以标准化。它将软件通信过程抽象化为四个抽象层，采取协议堆栈的方式，分别实现出不同通信协议。

1. TCP/IP体系结构

四层模型：网络接入层、互联网络层、传输层、应用层

其中，网络层包括IP协议、ICMP(因特网控制消息)协议、IGMP(因特网组管理)协议、ARR(地址解析)协议、RARP(反向地址解析)协议
传输层 包括TCP、UDP协议；应用层包括SMTP、FTP、DNS、HTTP等协议

2. TCP协议

2.1 特点

  • TCP是面向连接的，通信前需要建立TCP连接；
  • 提供可靠的服务
  • 只能点到点的通信，不提供广播或组播服务
  • 首部固定部分长度20字节，最长可达60字节
  • 面向字节流

2.2 三次握手

  • 客户端向服务器发送请求连接报文，其首部中的同步比特SYN置1，同时选择一个初始序号Seq=x;
  • 服务器收到请求，发回确认，ACK置1，确认序号为ACK=1，服务器此时的确认号应为x+1，服务器也发送和客户端连接的请求，在报文中将SYN置1，同时选择一个初始序号Seq=y；
  • 刻划断收到报文给服务器确认回复，确认序号为Seq=x+1，ACK=y+1。

2.3 四次挥手

  • 主机A完成数据传输后，将控制位FIN置1，提出停止TCP连接的请求
  • 主机B收到释放链接请求后，确认释放A到B的连接。此时连接处于半关闭状态 ，如果B继续发送数据，A仍然接受；
  • 主机B完成数据传输后，将控制位FIN置1，提出释放B到A的连接
  • 主机A确认释放连接，至此双方连接都已经释放

2.4 有限状态机

  • 三次握手时，客户端在发送最后一个ACK后进入ESTABLISHED状态，服务器在收到最后一个确认ACK时，也转为ESTABLISHED状态。
  • 四次挥手时，客户端先请求释放链接，发送FIN置1的报文后，转为FIN_WAIT_1状态，服务器收到FIN并发送确认ACK后，转为CLOSED_WAIT状态；客户端收到ACK后，由FIN_WAIT_1转为FIN_WAIT_2状态；服务器发送FIN置1报文请求释放连接，客户端收到后，客户端进入TIME_WAIT状态，等待报文段在网络中寿命的两倍时间，转为CLOSED状态，服务端收到ACK后，转为CLOSED状态。
