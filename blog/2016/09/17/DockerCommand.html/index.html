Docker 常用命令

发表于 2016-09-17 | 分类于 Cloud Computing ， Docker | 阅读次数

Docker中一些常用的命令。

1、常用命令

  • 查看Docker信息
    docker info
  • 查看本地主机的顶层（top-level）镜像
    docker images
    • top-level镜像是指 创建容器时使用的镜像或者是直接pull下来的镜像
  • 查看所有镜像 docker images -a
    • 可以说是列出了所有的可读层吧
  • 获取镜像
    sudo docker pull NAME[:TAG]
    • 默认情况下是从Docker Hub中下载相应最新版本的镜像,即从默认注册服务器registry.hub.docer.com中相应仓库中下载标记为latest的镜像;
    • 手动选择从其他注册服务器的仓库下载镜像，例如:从DockerPool的镜像源下载 sudo docker pull dl.dockerpool.com:5000/ubuntu
  • 删除镜像
    sudo docker rmi IMAGE[IMAGE...]
    • IMAGE可以是标签或者ID(部分前缀串)
    • 使用标签删除时，不会影响镜像文件，该命令只是删除了镜像多个标签中的一个，但当只有一个标签时，会彻底删除镜像
    • 使用ID删除时，会先删除指向该镜像的所有标签，然后删除镜像（注：当有该镜像创建的容器存在的时候，镜像文件默认是无法删除的）
  • 查找镜像
    sudo docker search TERM
    • 默认搜索Doucker Hub官方仓库中的镜像
  • 创建镜像
    • 基于已有镜像的容器
      sudo docker commit [OPTIONS] CONTAINER [REPOSITORY [:TAG]]
    • 基于本地模板导入
      sudo cat ubuntu-14.04-x86_64-minimal.tar.gz | docker import - ubuntu:14.04
    • 基于Dockerfile 创建（待补充！！！）
  • 导出和载入镜像
    docker commit gitlab(容器名称) gitlab-ubuntu:16.04(导出镜像名称)
    docker save -o gitlab-ubuntu.tar gitlab-ubuntu:16.04
    docker load --input gitlab-ubuntu.tar
    • 镜像ubuntu:14.04导出到当前目录下 ，命名为：ubuntu_test.tar
  • 上传镜像
    sudo docker push DockerHub的用户名/镜像名
    • 首次需要登录，以DockerHub的用户名和密码进行登陆
  • 创建容器
    sudo docker create -i -t <image-name> 如：sudo docker create -i -t ubuntu:14.04，返回创建的容器 ID
    • 该条指令为指定的镜像创建一个可读可写层，构成了一个新的容器，此时该容器并没有运行
  • 启动容器
    sudo docker start <container-id>
    • 为容器文件系统创建一个进程隔离层（每个容器有且只有一个进程隔离层）
  • 创建并启动
    sudo docker run -t -i ububtu /bin/bsah
    • 如果没有ubuntu这个镜像，会先从DockerHub上下载
    • -t 选项让Docker分配一个伪终端并绑定到容器的标准输入上，-i 让容器的标准输入保持打开
    • 默认启动的容器权限为root，但真实情况并没有足够的管理员权限， 此时的Docker 容器在权限方面并没有拥有宿主机上 root 相应的权限，可以通过指定参数--privileged=true，来获得真正的root权限。
  • 进入容器
    sudo docker exec -t -i 容器ID [ARG..]或 sudo docker attach 容器NAMES
    • 建议使用第一条命令，比如进入ubuntu镜像下的一个容器，并启动bash sudo docker exec -t-i 930fe /bin.bash
  • 退出容器
    ctrl+p ctrl+q
  • 删除容器
    sudo rm 容器ID
  • 查看已有容器
    sudo docker ps -a
  • 查看正在运行的容器
    sudo docker ps
  • 获取容器的输出信息
    sudo docker logs 容器ID
  • 导出容器
    sudo docker export 容器ID >导出文件名(例：sudo docker export 87816 >test_contain.tar)
  • 导入容器,成为镜像
    cat test_contain.tar | sudo docker import - REPOSITORY:TAG
  • 查看到容器的相关信息
    sudo docker inspect 容器id
  • 容器与主机间互传文件()
    从主机到容器：docker cp 文件名 容器名:容器内文件存放路径
    从容器到主机：docker cp 容器名:容器内文件存放路径 主机路径
    注：以上两条命令都是在主机内执行
    建议使用数据卷

2、区分save命令和export命令区别

  • Export命令用于持久化容器（不是镜像），导入导出的镜像会丢失历史数据
  • Save命令用于持久化镜像（不是容器），保存导入的镜像不会丢失历史数据，可以实现层回滚
