GitLab与Gogs的比较

发表于 2016-09-17 | 分类于 Git | 阅读次数

GitLab和Gogs都是类似GitHub的版本管理工具，GitLab是一个利用Ruby on Rails开发的开源应用程序，实现一个自托管的Git项目仓库，Gogs (Go Git Service) 是一款使用Go语言编写的，极易搭建的自助 Git 服务。

GirLab VS Gogs

对比 GitLab Gogs
大小 1.2G 89M
便捷性 安装较简单 安装部署简单
数据存储 默认postegresql存储，支持mysql，支持数据卷挂载 支持sqllite3（推荐）、mysql、postgresql，支持数据卷挂载
功能 支持分支保护、支持代码审查、代码片段、ruby on rails 、支持 SSH 以及 HTTP/HTTPS 协议、支持 SMTP、LDAP 和反向代理的用户认证、支持反向代理子路径、支持用户、组织和仓库管理系统、支持仓库和组织级别 Web 钩子（包括 Slack 集成）、支持仓库 Git 钩子和部署密钥、支持仓库工单（Issue）、合并请求（Pull Request）以及 Wiki、支持添加和删除仓库协作者、支持 Gravatar 以及自定义源、支持邮件服务、支持后台管理面板 资源占用少、跨平台、支持中文、支持活动时间线、支持 SSH 以及 HTTP/HTTPS 协议、支持 SMTP、LDAP 和反向代理的用户认证、支持反向代理子路径、支持用户、组织和仓库管理系统、支持仓库和组织级别 Web 钩子（包括 Slack 集成）、支持仓库 Git 钩子和部署密钥、支持仓库工单（Issue）、合并请求（Pull Request）以及 Wiki、支持添加和删除仓库协作者、支持 Gravatar 以及自定义源、支持邮件服务、支持后台管理面板、支持 MySQL >= 5.5.3、PostgreSQL、SQLite3 和 TiDB（实验性支持） 数据库、支持多语言本地化（14 种语言）
缺点 不能修改 clone 地址、仅支持linux 不支持分支保护、无法在同一个仓库内创建合并请求，不支持 Review Comments
